# 1	服务器部署项目代码
1. 创建配置虚拟机
2. 在终端上连接到内网：`ssh -J tiaoban@223.223.185.189 root@172.18.0.26      micl1202`
3. 通过 189 跳板机连接服务器：`ssh root@ip` （虚拟机 ip）
	1. 安装 docker
		1. 创建文件夹放 docker 镜像文件包：`mkdir packages   cd packages`
		2. 下载基础 docker 包：[Index of linux/ubuntu/dists/jammy/pool/stable/amd64/](https://download.docker.com/linux/ubuntu/dists/jammy/pool/stable/amd64/)
			1. 需要下载四个包：containerd、docker-ce-cli、docker-ce、docker-compose-plugin
			2. 在 packages 目录下下载（以 containerd 为例）：`wget https://download.docker.com/linux/ubuntu/dists/jammy/pool/stable/amd64/containerd.io_1.7.25-1_amd64.deb`
		3. 安装 docker：ls 查看当前文件夹的包 dpkg -i 进行下载 ----`dpkg -i *.deb`
	2. 安装 anaconda
		1. 下载 Miniconda 安装脚本：`wget https://repo.anaconda.com/miniconda/Miniconda3-latest-Linux-x86_64.sh`
		2. 运行安装脚本，按提示完成安装：`bash Miniconda3-latest-Linux-x86_64.sh`
		3. 重新加载 shell 配置文件，使 conda 命令立即生效：`source ~/.bashrc`
		4. 将 conda 初始化代码写入 ~/.bashrc，确保每次登录自动激活 base 环境：`conda init bash`
		5. 添加清华镜像源，加速 conda 包的下载，并显示 channel 地址
			1. `conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/free/`
			2. `conda config --set show_channel_urls yes`
4. 从跳板机上复制 docker 镜像
	1. 退出服务器，登录 189 跳板机（ctrl+d 即可）
	2. 将跳板机的文件 copy 到主机上
		1. 所有的镜像文件都放置在 czh/images，故切换到 czh 文件夹下
		2. 复制文件：`scp -r images root@ip :/root/packages/images`
		3. 还有一个向量模型文件放在根目录的 bjx 文件中（这是一个压缩包）
	3. 连接到服务器，将这几个镜像文件进行安装
		```
		docker load -i es.tar
		docker load -i kibana.tar
		docker load -i memcached.tar
		docker load -i mongo.tar
		docker load -i nginx.tar
		docker load -i postgres.tar
		docker load -i python.tar
		docker load -i rabbitmq.tar
		```
	4. 用 docker images 进行查看，验证所有镜像已加载
5. 克隆代码仓库（放在服务器根目录，/root 下）
	1. `git clone https://gitee.com/ustb-mge_1/mgedata.git` 登入自己 gitee 的网站的账号和密码
	2. 复制向量模型文件：将向量模型文件放在 mgedata/docker/中
		1. 第一种：退回到跳板机，`scp bjx root@ip:/root/mgedata/docker/`
		2. 第二种：`scp -o "ProxyJump= tiaoban@223.223.185.189 :22" -r ~/bjx root@ip:/root/mgedata/docker/`
	3. 连接到服务器，这是一个压缩包进行解压：`tar -xf bjx`
	4. 切换分支：现阶段用 portal-migrate-px-test 这个分支来进行开发
		1. git branch 进行分支的查看
		2. git checkout portal-migrate-px-test 创建新的分支
6. swap 内存增加到 32GB
	1. **检查 swap 设置**: free -h
	2. **检查现有的 swap 文件或分区**: sudo swapon –show
	3. **关闭现有的 swap**: sudo swapoff -a
	4. **创建新的 swap 文件**: sudo dd if=/dev/zero of=/swapfile bs=1M count=32768
	5. **设置交换文件权限**: sudo chmod 600 /swapfile
	6. **将文件格式化为 swap 空间**: sudo mkswap /swapfile
	7. **启用新的 swap 文件**: sudo swapon /swapfile
	8. **检查新的 swap 设置**: free -h
	9. （可选，了解即可）要使 swap 文件在系统重启后自动启用，需要将它添加到 `/etc/fstab ` 文件中。在文件末尾添加以下行：`/swapfile none swap sw 0 0`
7. 基于 Docker 构建和启动项目服务（分为部署版本和开发服版本,构建都在 mgedata-- 项目根目录下进行）
	1. 数据库部署版本
		1. 构建一个名为 `mge-base:latest` 的 Docker 镜像：`docker build -t mge-base:latest -f docker/mge-base.Dockerfile .`
		2. 使用 `docker-compose-dev-deploy.yml` 编排文件，在后台启动所有服务容器：`docker compose -f docker/docker-compose-dev-deploy.yml up -d --build`
		3. 查看容器启动情况：`docker ps`
	2. 数据库开发版本（了解即可）：`docker compose -f docker/docker-compose-dev-local.yml up -d –build` 开发服总共有四个容器
8. 在跳板机上开设公网转发端口
	1. 查看所有端口转发：`ps aux|grep ssh`
	2. 可以查看服务器是否被 ssh 映射：`ps aux|grep ip`
	3. ssh 映射：`ssh -f -N –g -L 端口号：ip：22 root@ip` （22 端口用于 ssh 的开发服，80 端口是部署服）
	4. 登入跳板机管理员账号之后，开放端口并重启防火墙 -- 管理员账号：su 密码： m1cl@1202
		```
		firewall-cmd --zone=public --add-port=7080/tcp --permanent
		firewall-cmd --reload
		```
9. 进入内网 IP
	1. 开发服安装 python 环境（部署服不需要进行这步操作，自带的 Docker 容器中已存在）（了解即可）
		1. 创建一个 mge 的虚拟环境用于开发服：`conda create -n mge python=3.10 `
		2. 在 mgedata/docker 下的 mge-base.Doockerfile 有 docker 容器所需要的依赖安装：`Cat mge-base.Dockerfile` 进行查看
		3. 将 pip 的默认镜像源设置为阿里云镜像，安装依赖
			```
			Pip config set global.index-url https://mirrors.aliyun.com/pypi/simple/&&pip install -U setuptools wheel pip 
			Pip install poetry ipython 
			Poetry install –no-interactionn –no-ansi
			```
		4. 输入上面三条安装指令之后会有报错，原因在于某些包装不上，需要注释掉
			1. 在 mgedata 这个目录下 ：`vim pyproject.toml` 进行查看所有的依赖环境库，注释掉 uwsgi 这个库
			2. 之后进行安装依赖 poetry lock：`Poetry install –no-interaction –no-ansi`
		5. ==安装完依赖之后进行如下系统设置（部署服也需要）==
			```Bash
			sudo sed -i '/^GRUB_CMDLINE_LINUX=/ s/"$/ cgroup_enable=memory swapaccount=1"/' /etc/default/grub
			# 临时设置
			sudo sysctl -w vm.max_map_count=262144
			# 永久设置
			vim /etc/sysctl.conf  ## 添加vm.max_map_count=262144
			# 应用
			sudo sysctl -p
			# 验证
			sysctl vm.max_map_count
			```
		6. 之后还需要在 mgedata 目录中 (root/mgedata/mgedata) 复制 local_setting-dev_example.py 一份改名为 local_setting-dev.py：`cp local_setting_dev_example.py local_setting_dev.py`
		7. 激活虚拟环境（命令提示符前有虚拟环境名称，默认应该是自动激活），运行 `python manage.py migrate && python manage.py init_dev` 等待执行完成。成功后会显示 超级管理员 的用户名和密码，同时会创建测试用的领域分类和项目、课题。
		8. 运行 Django 项目: `python manage.py runserver`
	2. 部署服也需要进行系统设置
	3. 部署服启动 Django 项目
		1. 数据迁移：`docker exec -it 容器名 poetry run python manage.py migrate`
		2. 执行项目的初始化脚本
			```Bash
			# 进入容器并启动Django
			docker exec -it 容器名 bash
			poetry env info
			/root/.cache/pypoetry/virtualenvs/mgedata-E3x33NbX-py3.10/bin/python manage.py init_dev
			# 一句命令即可实现
			docker exec -it 容器名 poetry run python manage.py init_dev
			```
# 2	代码更新
1. 进入项目目录，一般是根目录下的 mgedata 目录下（管线钢是根目录下 guanxiangang/mgedata）
	1. 执行 git 命令 git branch，检查当前分支，如果不是，则先执行 git pull 后，执行 git checkout 分支名来切换分支。
	2. 分支正常，也执行 git pull，输入自己的 gitee 账号密码，然后拉取最新代码。
	3. 重启 docker ，命令（先删容器 → 再重新构建 → 再全部启动）：`docker compose -f docker/docker-compose-dev-deploy.yml down && docker compose -f docker/docker-compose-dev-deploy.yml up -d --build
2. 如果 pull 出错，需要 git stash 之后再进行以上操作
3. 如果在添加新的文件之前已经构建好新的 Docker 容器，删掉 celery nginx mge-base 这三个容器重新构建即可
# 3	重新拉取代码（更改分支）部署
1. 停掉容器，删除容器，删除网络
	```
	docker compose -f docker/docker-compose-dev-deploy.yml down
	```
2. 重新拉取代码吗，更新分支
3. 复制向量模型文件
4. **重新构建镜像**：`docker build -t mge-base:latest -f docker/mge-base.Dockerfile .`
5. **重新启动服务**：`docker compose -f docker/docker-compose-dev-deploy.yml up -d --build`
